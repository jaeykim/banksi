# x402: HTTP 402 Payment Required

## Concept

HTTP 402 is a status code reserved for "Payment Required." The **x402 protocol** extends this by defining a machine-readable response format that tells clients (including AI agents) exactly how to pay for access to a resource using cryptocurrency.

This enables **programmatic micropayments** — an AI agent hits an API, gets a 402 response, pays the required stablecoin amount on-chain, and retries with proof of payment. No human intervention needed.

## API Reference

### 402 Response Format

When a protected endpoint is accessed without payment:

```json
{
  "status": 402,
  "paymentRequired": {
    "scheme": "x402",
    "version": "0.1",
    "description": "Access premium market data. Costs $0.01.",
    "merchant": "seoul-coffee",
    "merchantName": "Seoul Coffee",
    "amount": 0.01,
    "currency": "USD",
    "chains": [
      {
        "id": "ethereum",
        "name": "Ethereum",
        "tokens": [
          { "id": "clxx...", "symbol": "USDT" },
          { "id": "clxx...", "symbol": "USDC" }
        ]
      }
    ],
    "payUrl": "http://localhost:3001/api/x402/pay",
    "howToPay": [
      "1. POST /api/x402/pay with { merchantSlug, chainId, tokenId, amount }",
      "2. Send stablecoin to the returned address.",
      "3. Wait for blockchain confirmation.",
      "4. Retry with header: X-Payment: <paymentId>"
    ]
  }
}
```

### `POST /api/x402/pay`

Create a payment for an x402 paywall.

**Request:**
```json
{
  "merchantSlug": "seoul-coffee",
  "chainId": "ethereum",
  "tokenId": "clxx...",
  "amount": 0.01
}
```

**Response (201):**
```json
{
  "paymentId": "cmna...",
  "address": "0xABC...",
  "amountExpected": "0.01",
  "tokenSymbol": "USDT",
  "chainName": "Ethereum",
  "expiresAt": "2026-03-28T07:30:00.000Z"
}
```

### Demo Endpoint

`GET /api/x402/demo` — Protected endpoint requiring $0.01 payment.

**Without payment:** Returns 402 with payment instructions.

**With valid payment:**
```bash
curl -H "X-Payment: <paymentId>" http://localhost:3001/api/x402/demo
```
```json
{
  "data": {
    "message": "Payment verified! Here is your premium content.",
    "marketData": { "btcPrice": 97542.50, "ethPrice": 3821.30 }
  }
}
```

## Protect Your Own Endpoint

```typescript
import { requirePayment } from '@/lib/x402/middleware';

export async function GET(request: NextRequest) {
  const paywall = await requirePayment(request, {
    merchantSlug: 'your-store',
    amount: 0.05,
    description: 'Access this API for $0.05',
  });

  if (paywall) return paywall; // Returns 402

  // Payment verified — serve content
  return NextResponse.json({ data: '...' });
}
```

## For AI Agents (MCP)

The Banksi MCP server includes an `x402_pay` tool. When an agent encounters a 402 response, it can:

1. Read the `paymentRequired` object from the 402 body
2. Call `x402_pay` with the merchant, chain, token, and amount
3. Receive address and payment instructions
4. After on-chain payment, retry with `X-Payment` header

See [MCP Server docs](./mcp-server.md) for setup.

## Use Cases

- **API monetization** — Charge per-request for premium data feeds
- **AI agent payments** — Agents autonomously pay for API access
- **Content paywalls** — Gate premium content behind micropayments
- **Rate limiting alternative** — Pay-per-use instead of API keys
